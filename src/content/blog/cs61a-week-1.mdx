The first week keeps repeating one rule: use small functions and keep the boundaries obvious.

### What matters

1. Function names should say what the value means.
2. Arguments should stay small enough to inspect by hand.
3. Recursive steps only make sense when the base case is easy to see.

```python
def square(x):
    return x * x

def sum_of_squares(a, b):
    return square(a) + square(b)
```

The code itself is ordinary. The point is to avoid hiding the shape of the problem.

This note keeps the deployment path explicit. Build once, serve the generated files, and keep the web server out of the app logic.

### Checklist

1. Run the sync step before the build.
2. Copy `dist/` to the server over SSH or rsync.
3. Let Nginx serve the static files directly.

The reusable checklist lives here:

- [server-checklist.txt](/files/server-checklist.txt)

```nginx
server {
    listen 80;
    server_name example.com;

    root /var/www/blog/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

That setup keeps the runtime surface small and makes rebuilds predictable.
